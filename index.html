İdeaShop API’ye Merhaba!

İdeashop Api REST mimarisini ve OAuth2 standartlarını kullanmaktadır. Api hizmetimizden en yüksek verimle yararlanmak için öncelikle bu iki standarda aşina olmanız gerekmektedir.

RESTful Api Nedir?

Kısaca API (Application Programming Interface), herhangi bir uygulamanın belli işlevlerini diğer uygulamalarında kullanabilmesi için oluşturulmuş bir modüldür. REST (Representational State Transfer) ise bu modülün kullanımını oldukça kolaylaştıran bir mimaridir. REST’i diğer mimarilerden ayıran belli başlı özellikler şunlardır:

  • # Client-server iletişimiyle ilgili bir mimaridir.
  • # SOAP, RPC gibi kompleks mimarilerle sağlamak yerine, HTTP protokolü üzerinden sağlar.
  • # SOAP, RPC'nin aksine basit ve hafiftir.
  • # Esnek olup, SOAP gibi keskin standartları yoktur.
  • # SOAP ile en büyük farkı, SOAP gibi bizi proxy kullanmaya, bir WSDL'e zorlamaz
  • # Global bir mimari olup entegre etmesi kolaydır.
  • # Input ve output verilerini JSON formatında işlemektedir.

REST istek Tipleri ve Anlamları

GET: Veri listeleme - veri görüntülemek için kullanılır.
POST: Veri eklemek için kullanılır.
PUT: Veriyi Güncelleme isteği olarak kullanılır.
DELETE: Veriyi silmek için kullanılır.

İdeashop Api sadece bu dört istek tipini kullandığı için dökümantasyonda diğer istek tiplerine değinmiyoruz. REST mimarisi ile ilgili ayrıntılı bilgi edinmek için buraya tıklayın.

OAuth2 nedir?

İdeashop Api OAuth2 Kimlik Doğrulama Standardını kullanmaktadır. Peki nedir bu kimlik doğrulama ve neden biz OAuth2 standardını kullanıyoruz?

Kimlik doğrulama işlemi sayesinde Api’mize kimlerin hangi seviyede erişimi olacağını kontrol edebiliriz. OAuth2 ise bu işlemi en güvenli ve kullanımı kolay halde yapmamıza yaramaktadır. OAuth2 sayesinde yönetici olan kullanıcı, sınırlı yetkileri olan kullanıcılar oluşturup bu kullanıcılara kendi isim ve şifrelerini kullanarak Api erişim izni verebilir. Yani siz bir İdeashop site yöneticisi olarak panelinizde sadece sizin izin verdiğiniz bilgilere erişimi olan kullanıcılar oluşturup onlara yine sizin izin verdiğiniz sınırlar çerçevesinde Api kullanım yetkisi verebilirsiniz. Böylece hem veri güvenliğiniz sağlanmış olur hem de harici kullanıcılar üzerinden e-ticaret sitenizi daha kullanışlı hale getirebilirsiniz.

OAuth2 standardı ile ilgili daha ayrıntılı bilgi edinmek için buraya tıklayın.
